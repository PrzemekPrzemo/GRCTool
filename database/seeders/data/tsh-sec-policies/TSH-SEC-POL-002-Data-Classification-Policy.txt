DATA CLASSIFICATION POLICY
BY PRZEMYSŁAW SZULECKI, CSO
Defines TSH's four-level information classification scheme (RESTRICTED → PUBLIC) and minimum handling requirements for each level. Client scheme mappings are maintained in POL-002-ANX annexes.
TSH-SEC-POL-002  |  v2.2  |  2026-07-13  |  CSO — Przemysław Szulecki
# 1. Purpose & Scope
This policy defines TSH's information classification scheme and minimum handling requirements for each level. It applies to all information created, received, processed and stored by TSH — including client data. Correct classification is the foundation of proportionate data protection.
# 2. Classification Levels
Client data is CONFIDENTIAL by default and may only be reclassified upward (to RESTRICTED), never downward. When in doubt — classify higher.
# 3. Classification Rules
## 3.1 Default Classification
All information created within TSH projects is classified CONFIDENTIAL by default unless explicitly reclassified by the data owner.
## 3.2 Always RESTRICTED (regardless of context)
Authentication credentials — passwords, API keys, tokens, certificates, private keys
Personal data of natural persons (GDPR/PDPL definition) — including biometric and health data
Client source code and proprietary algorithms
Vulnerability reports, penetration test results, security audit findings
Legal documents and regulatory correspondence, dispute-related materials
Financial data — client revenues, budgets, payroll
## 3.3 Labelling
RESTRICTED and CONFIDENTIAL documents must carry a visible classification label. Digital documents: document metadata and header/footer label. Emails: [RESTRICTED] or [CONFIDENTIAL] in subject line. Code comments: // RESTRICTED: description.
## 3.4 Handling by Level
Credentials (passwords, API keys, tokens) must NEVER be stored in code repositories, emails, chat tools, or documents. Use 1Password exclusively. Violations are treated as P2 security incidents.
## 3.5 Client Scheme Mapping
Where a client operates its own classification scheme, TSH maps it to internal levels in a dedicated POL-002-ANX-[letter] annex. Absent an annex, client data defaults to CONFIDENTIAL. Client labels are neither removed nor downgraded.
