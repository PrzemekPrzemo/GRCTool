ACCESS CONTROL POLICY
BY PRZEMYSŁAW SZULECKI, CSO
This policy defines how access to TSH systems is granted, managed, reviewed, and revoked. It covers identity management, MFA, privileged access, and the full provisioning lifecycle.
TSH-SEC-POL-003  |  v2.1  |  2025-01-01  |  CSO — Przemysław Szulecki
# 1. Purpose & Scope
This policy defines how access to TSH systems and data is granted, managed, reviewed, and revoked. It applies to all TSH personnel, contractors, and third parties.
# 2. Identity & Access Principles
Least privilege — access granted only to what is needed for the role
Need to know — data access justified by business need
Separation of duties — no single person controls complete sensitive processes
Zero trust — no implicit trust based on network location
Just-in-time — privileged access activated on demand, not permanently assigned

# 4. MFA Requirements
Multi-factor authentication is mandatory for all accounts without exception.
SMS OTP is NOT permitted as sole authentication factor for any account. It may only be used as a last-resort emergency fallback with prior CSO approval.
# 5. Privileged Access Management
## 5.1 Entra ID PIM (Just-in-Time)
No permanent privileged role assignments — all elevated roles via PIM activation
Maximum activation duration: 4 hours. Requires business justification.
All PIM activations logged and reviewed weekly by IT Lead
Emergency access (break-glass) accounts: two accounts, stored offline, CSO and IT Lead only
## 5.2 Privileged Actions Requiring PIM
Entra ID Global Admin, Privileged Role Admin, Security Admin
Google Workspace Super Admin
GitHub Enterprise Owner
IRU (Apple MDM) admin actions — device wipe, policy push, enrollment changes
Production environment access (client systems)
# 6. Access Provisioning & Deprovisioning
## 6.1 Onboarding
Accounts provisioned T-2 business days before start date. IT receives HR notification via automated workflow. Standard role profile applied; additional access requires line manager approval in Jira ticket.
## 6.2 Offboarding
# 7. Access Reviews
All user accounts: quarterly review — IT Lead coordinates, managers confirm
Privileged accounts: monthly — CSO reviews PIM activation logs
Service accounts: quarterly — IT Lead validates each account's active use
External/contractor accounts: at project milestones and at engagement end
Unconfirmed accounts in access review are suspended after 7 calendar days of non-response. Access is not deleted until manager explicitly confirms removal.
