SECURITY LOGGING & MONITORING POLICY
BY PRZEMYSŁAW SZULECKI, CSO
No-SIEM monitoring architecture (Entra ID + MDE + Google + GitHub), three-layer DLP (Google Admin + MDE + IRU MAM), email authentication (SPF/DKIM/DMARC), log retention, and alert triggers. DLP config: CFG-308.
TSH-SEC-POL-014  |  v2.2  |  2025-01-01  |  CSO — Przemysław Szulecki
# 1. Purpose & Scope
This policy defines TSH's security monitoring architecture, logging requirements, DLP controls, and alert triggers. TSH uses a native-platform monitoring approach — see Section 2. DLP configuration details are in CFG-308.
# 2. Monitoring Architecture — No-SIEM Decision
TSH has made an explicit, documented decision not to operate a dedicated SIEM. This is appropriate for a cloud-native, infrastructure-free operational model. Equivalent coverage is achieved through four integrated platforms:
This no-SIEM decision is documented in REG-001 (Risk Register). Annual review confirms continued appropriateness. If TSH starts hosting own infrastructure, SIEM evaluation is triggered automatically.
# 3. Data Loss Prevention (DLP)
TSH implements DLP across three layers. Configuration details and step-by-step setup: CFG-308 (DLP Configuration Guide).
# 4. Email Authentication
TSH enforces email authentication for the tsh.io domain to prevent spoofing and phishing using our domain. Configuration details: CFG-301 Section 6.

# 6. Alert Triggers — Immediate CSO Notification
Admin account sign-in from new country or outside 08:00–22:00 CET
PIM activation for Global Admin, Privileged Role Admin, Security Admin
IRU device wipe/erase command executed (who, when, which device)
MDE: malware detection or ransomware behaviour
MDE DLP: RESTRICTED data copied to USB or unapproved cloud
GitHub: secret pushed to any repository
Google Workspace: government-backed attack warning
Google DLP: RESTRICTED-labelled file shared externally
Entra ID Identity Protection: HIGH risk user or sign-in
5+ failed MFA attempts on any account within 10 minutes

[TABLE]
Layer | Platform | What it controls
Email & Drive DLP | Google Workspace Admin — Security → DLP | Content scanning: RESTRICTED/CONFIDENTIAL labels, credit cards, IBAN, PII patterns. Actions: block external share (RESTRICTED), warn (CONFIDENTIAL), alert IT Lead.
Endpoint DLP | Microsoft Defender for Endpoint (MDE) | File operations on all managed devices (macOS via IRU, Windows via Intune): USB copy, upload to unapproved cloud, print of sensitive content. RESTRICTED violations alert IT Lead immediately.
Mobile/BYOD DLP | IRU App Protection Policy (Apple MAM) | BYOD iPhones: blocks copy-paste from managed apps to personal apps; blocks screen capture; selective wipe on offboarding without touching personal data.
